## Learning Objective

Practice sequencing and spatial logic by completing a simple bridge from left to right with large draggable blocks.

## Avril Fit

- Reading required: none
- Main interaction: drag blocks into matching silhouettes
- Learning target: sequencing and spatial logic
- Theme hook: original puppy helper and cozy bridge-building play
- Session length: 3-5 minutes
- Parent help needed: medium

## Game Rules

- Show an incomplete bridge with 3-5 obvious missing block slots.
- Show a small set of large draggable bridge blocks.
- Each block should have a clear matching silhouette or shape cue.
- Correct placement snaps the block into the bridge and gives gentle visual/audio feedback.
- Incorrect placement should drift back or softly bounce without punishment.
- Completion starts a short celebration: the puppy crosses the bridge and collects a colorful balloon.

## Core Loop

1. Present a bridge with a few missing pieces and matching loose blocks.
2. Avril drags a block toward a matching gap.
3. The block snaps into place when close enough.
4. The next gap remains visually obvious.
5. Once every gap is filled, the puppy crosses and the game celebrates.
6. A replay/reset affordance lets the loop restart without reading.

## Screens / States

- Landing menu card for Little Builder Bridge.
- Game scene with bridge, puppy helper, loose blocks, and target silhouettes.
- Correct placement feedback state.
- Soft wrong-placement feedback state.
- Completion celebration state.
- Replay/menu state.

## Assets Needed

- Original puppy helper, not based on any existing show character.
- Cozy bridge environment with simple, toddler-friendly shapes.
- 3-5 bridge block pieces with distinct silhouettes.
- Balloon reward asset.
- Menu preview image after implementation is visually stable.

## Audio / Feedback

- Audio is optional and must unlock after first user interaction.
- Use short local SFX only; no runtime TTS or network audio.
- Feedback should be positive and gentle.
- No text or spoken instruction may be required to understand the loop.

## Accessibility

- Large touch targets for mobile portrait.
- Forgiving drop zones that accept near misses.
- No timed pressure.
- No harsh failure state.
- No reading-dependent instruction.
- Visual contrast should make bridge gaps and draggable blocks easy to distinguish.

## Implementation Notes

- Use Phaser 3 + Vite + TypeScript.
- Keep the game browser-playable and offline-friendly after load.
- Do not add accounts, telemetry, analytics, cloud persistence, or child data collection.
- Use original inspired-by themes only. Do not use Netflix, Vera, Gabby, direct characters, logos, art, music, or story references.
- Keep implementation blocked until `docs/design/issue-10-little-builder-bridge.md` exists, unless Ramon explicitly waives the design step.
