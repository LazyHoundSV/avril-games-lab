## Learning Objective

Help Avril practice one-to-one matching through large color cues first, then simple shape cues once the interaction pattern is clear.

## Avril Fit

- Reading required: none
- Main interaction: drag and match
- Learning target: one-to-one matching, colors, simple shape recognition
- Theme hook: gentle magical garden with original flower gates and pet visitors
- Session length: 2-4 minutes
- Parent help needed: low

## Game Rules

- Show 2-3 garden gates with oversized lock faces.
- Each lock has a clear visual cue: start with color-only, then add simple shapes such as circle, star, and square.
- Tokens are large, chunky, and draggable.
- Correct drops snap onto the lock, open the gate, and reveal a friendly original garden visitor.
- Incorrect drops gently bounce back without harsh feedback or failure state.
- The main loop must remain playable without reading, spoken instructions, accounts, telemetry, analytics, child data collection, cloud persistence, or external publishing.

## Core Loop

Avril drags matching tokens onto garden gate locks. Each successful match opens one gate and lets a cat, puppy, or butterfly friend enter the garden. After all gates open, the scene celebrates with soft color sparkles around a tiny fountain.

## Screens / States

- Landing menu card for `Garden Gate Locks`.
- Game intro state with obvious draggable tokens and visible target locks.
- Round state with 2-3 gates and matching tokens.
- Correct placement feedback with gate opening and friend entering.
- Gentle incorrect placement feedback with token return.
- Completion state with friends gathered around the fountain plus replay/menu controls.

## Assets Needed

- Garden background with 2-3 gate positions.
- Closed and open gate visuals.
- Lock faces for color-only and color-plus-shape cues.
- Chunky draggable tokens matching the lock cues.
- Original cat, puppy, and butterfly visitor assets.
- Fountain and soft sparkle celebration visuals.
- Landing menu preview image.

## Audio / Feedback

- Optional short placement chime after first user gesture.
- Optional soft gate-open sound.
- Optional completion applause or sparkle sound.
- Audio must not be required for comprehension.

## Accessibility

- Large touch targets and forgiving drop zones.
- Strong color contrast and distinct simple shapes.
- No text-dependent instructions.
- No timed pressure.
- Calm, positive feedback with no punitive failure state.

## Implementation Notes

- Use Phaser 3 + Vite + TypeScript.
- Keep gameplay state local to the browser session.
- Prefer layout helpers with unit tests for desktop and mobile geometry.
- Add the game to the existing landing menu and preserve existing game routes.
- Use original inspired-by garden/pet visuals only; do not use Netflix, Vera, Gabby, direct characters, logos, art, music, or story references.

## Validation Plan

- Loads on desktop browser.
- Loads on mobile viewport.
- Drag/tap interactions work.
- No reading required for the main loop.
- No external IP assets are used.
- No child data collection is introduced.
