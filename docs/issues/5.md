# Issue Spec - Kitty Room Builder

## GitHub

- Issue: https://github.com/LazyHoundSV/avril-games-lab/issues/5
- Status: design ready
- Owner: Hawks
- Pixie spec: `docs/design/issue-5-kitty-room-builder.md`

## Learning Objective

Support toddler-friendly spatial logic and object matching by asking Avril to place large cozy room objects into obvious matching silhouettes.

## Avril Fit

- Reading required: none
- Main interaction: drag and place
- Learning target: spatial logic and object matching
- Theme hook: original cozy cat room / dollhouse-like building
- Session length: 3-5 minutes
- Parent help needed: low

## Game Rules

- Show a small cozy room with 4-5 large empty silhouettes.
- Show draggable room objects such as a bed, rug, ball, window, and bowl.
- Each object has one obvious matching silhouette.
- Correct placements snap into place and trigger a gentle happy response.
- Incorrect placements return softly without harsh failure feedback.
- The round completes when all objects are placed.

## Core Loop

Avril sees a room with visible ghost outlines, drags each object into its matching spot, and fills the room one object at a time. As the room becomes complete, the original kitten helper reacts with calm, positive animation.

## Screens / States

- Landing menu card for Kitty Room Builder.
- Game start state with the room, silhouettes, and draggable objects.
- Placement feedback state for correct and incorrect drops.
- Completion state with the kitten helper entering the finished room.
- Replay/menu state using icon-first controls.

## Assets Needed

- Cozy room background with clear floor/wall separation.
- Large silhouettes for bed, rug, ball, window, and bowl.
- Matching draggable object sprites.
- Original kitten helper character.
- Simple celebration accents such as stars, hearts, or soft room glow.

## Audio / Feedback

- Soft placement sound for correct drops.
- Gentle bounce-back sound for incorrect drops.
- Calm completion chime.
- No voiceover or text-dependent instruction required for the main loop.

## Accessibility

- Large touch targets for every draggable object and silhouette.
- High contrast between objects, silhouettes, and background.
- Forgiving drop zones around silhouettes.
- No timed pressure.
- No reading required for the main loop.
- Avoid visual clutter; keep one primary action visible.

## Implementation Notes

- Build with Phaser 3 + Vite + TypeScript.
- Keep the route reachable from the main landing menu.
- Use original art direction and avoid copyrighted Vera, Gabby, Netflix, direct characters, logos, art, music, or story references.
- Do not add accounts, telemetry, analytics, child data collection, cloud persistence, or external publishing.
- Wait for Pixie's UI/UX spec before implementation unless Ramon explicitly waives the design step.
- Do not start this build while another approved game is actively in progress.

## Validation Plan

- Loads on desktop browser.
- Loads on mobile viewport.
- Drag/tap interactions work.
- No reading required for the main loop.
- No external IP assets are used.
- No child data collection is introduced.
