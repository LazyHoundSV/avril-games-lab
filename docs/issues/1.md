# Issue 1 - Color Basket Garden

- Owner: Hawks

## Learning Objective

Help Avril practice color recognition and simple classification by sorting colorful garden objects into matching baskets.

## Avril Fit

- Reading required: none
- Main interaction: drag and sort
- Learning target: colors and classification
- Theme hook: magical garden collecting
- Session length: 2-4 minutes
- Parent help needed: low

## Game Rules

- Show a small set of large, colorful objects such as flowers, fruits, and leaves.
- Show matching baskets using strong visual color cues, not written labels.
- Avril drags each object into the basket with the same color.
- Correct matches should snap into the basket and trigger a clear positive animation.
- Incorrect matches should gently bounce back or stay available without harsh failure feedback.
- Complete the round after all visible objects are sorted.

## Core Loop

1. Present 2-3 baskets and a small group of garden objects.
2. Avril drags an object toward a basket.
3. The game checks whether the object's color matches the basket.
4. Correct placements fill the basket and add garden sparkle/progress feedback.
5. Incorrect placements return gently so Avril can retry.
6. When all baskets are filled, a friendly helper animal celebrates.

## Screens / States

- Start state: visual-only play prompt with baskets and objects already visible.
- Sorting state: draggable objects, basket drop zones, and progress feedback.
- Correct match state: snap, sparkle, basket fill, gentle sound cue if audio is available.
- Incorrect match state: soft bounce/return with no penalty or scary feedback.
- Completion state: garden celebration with helper cat/dog and restart option.

## Assets Needed

- Garden background or play area.
- 2-3 large color baskets.
- Garden objects in matching colors, likely flowers, fruits, and leaves.
- Friendly helper animal for celebration.
- Simple sparkle / confetti / glow effects.
- Optional small icon-only restart control.

## Audio / Feedback

- Audio should be optional and non-essential.
- Correct matches can use a short pleasant chime.
- Incorrect matches should use either no sound or a very soft neutral sound.
- Completion can use a short celebration sound.
- All feedback must remain understandable visually.

## Accessibility

- No text-dependent child-facing instructions.
- Large touch targets for objects and baskets.
- High contrast between target colors.
- Avoid relying on subtle shades that are hard to distinguish.
- Keep on-screen object count low to reduce overwhelm.
- Support pointer and touch input.

## Implementation Notes

- Technical default is Phaser 3 + Vite + TypeScript.
- Keep the first version browser-playable and offline-friendly.
- No accounts, telemetry, child data collection, analytics, cloud persistence, or backend.
- Use original art and themes only; no Netflix, Vera, Gabby, direct characters, logos, art, music, or story references.
- Build as the only active approved game until it is done or explicitly paused.

## Validation Plan

- Loads on desktop browser.
- Loads on mobile viewport.
- Drag interactions work with mouse and touch.
- No reading is required for the main loop.
- Correct object-to-basket matches complete successfully.
- Incorrect matches are gentle and retryable.
- Completion state appears after all objects are sorted.
- No external IP assets are used.
- No child data collection is introduced.
